# Beijing PM2.5 Air Quality Prediction 

Проект по анализу временных рядов и прогнозированию концентрации мелкодисперсных частиц PM2.5 в Пекине. В проекте используются объединенные данные Института окружающей среды Университета Цинхуа (Tsinghua University) и системы мониторинга качества воздуха Посольства США в Пекине.

## Описание проекта

Цель проекта-построение моделей машинного обучения или глубокого обучения (Deep Learning) для предсказания уровня загрязнения воздуха (PM2.5) на основе метеорологических данных и исторических показателей загрязнения. Данный проект решает задачу регрессии на временных рядах (Time Series Regression).

## Описание датасета

Набор данных содержит ежечасные метеорологические показатели и уровень загрязнения воздуха в Пекине за период с **1 января 2010 года по 31 декабря 2014 года**.

### Ключевые признаки (Features):
*   `No`: Номер строки (индекс)
*   `year`, `month`, `day`, `hour`: Временные метрики (год, месяц, день, час)
*   `pm2.5`: Концентрация частиц PM2.5 (мкг/м³) — *целевая переменная*
*   `DEWP`: Точка росы (Dew Point, °C)
*   `TEMP`: Температура воздуха (°C)
*   `PRES`: Атмосферное давление (гПа)
*   `cbwd`: Комбинированное направление ветра (категориальный признак)
*   `Iws`: Накопленная скорость ветра (м/с)
*   `Is`: Накопленная продолжительность снегопада (в часах)
*   `Ir`: Накопленная продолжительность дождя (в часах)

> **Важно:** Оригинальный датасет содержит пропуски (`NaN`) в столбце `pm2.5`, которые требуют предварительной обработки (интерполяция или удаление) перед обучением моделей.

## Стек технологий

*   **Язык:** Python 3.x
*   **Анализ данных:** Pandas, NumPy
*   **Визуализация:** Matplotlib, Seaborn, Statmodels (для интерактивных графиков временных рядов)
*   **Машинное обучение:** Scikit-learn
*   **Глубокое обучение:** TensorFlow


## Результаты работы

*   Проведен разведочный анализ данных (EDA): выявлена сильная сезонность (пики загрязнения зимой из-за отопительного сезона) и зависимость PM2.5 от направления и скорости ветра.
*   Успешно обработаны пропущенные значения во временном ряду.
*   Для прогнозирования протестированы модели **[LSTM / LSTM+Decocder(seq2seq]**.
*   Финальная модель достигла метрики **RMSE = [25.26]** на тестовом периоде.
